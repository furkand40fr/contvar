ContVAR: Contrastive Graph Representation Learning for Protein Variant Function Prediction

Motivation

From "damaging vs. neutral" to which function is lost

Single amino-acid variants (SAVs) drive disease mechanisms, oncogenic transformations, and drug response, but experimental characterization is slow — most variants remain annotated only as variants of uncertain significance.

Existing variant-effect predictors (SIFT, PolyPhen-2, CADD, ESM-1v) compress mutation impact into a scalar deleteriousness score, while GO-annotation models (DeepGOPlus, DeepFRI, DeepGraphGO) operate on wild-type proteins. Neither side answers the clinically and biotechnologically critical question: which specific molecular function is gained or lost due to a SAV?

ContVAR targets this gap with a two-module AI system:

  • Encoder — self-supervised graph contrastive model that turns a residue graph into a mutation-aware 256-D embedding.
  • Decoder — supervised multi-label GO classifier on top of those embeddings; variant effect is read off as Δscore = score(WT) − score(variant) per GO term.

Data & Featurization

Two complementary supervision tracks

Phase 0 — GO. Pre-built PyG .pt graphs from Swiss-Prot are paired with three semantic-similarity TSVs (Molecular Function, Biological Process, Cellular Component). Positives use similarity ≥ 0.8; negatives use ≤ 0.2 (sampled from low/mid bins for hardness). Triplets whose proteins are missing in the prebuilt graph index are dropped.

Phase 1 / 2 — DMS. 97 protein families curated in a ProteinGym-style organization, split 93 / 2 / 2 at the family level to avoid leakage. Each family contributes one wild-type anchor plus multiple positives and multiple negatives. CIFs are parsed via MMCIFParser, converted to PyG graphs through Graphein, and cached as processed/<protein>.pt.

Node features. Each residue is represented by

Per-residue input xi = [ AA-OneHoti  ‖  ESM2i ]  ∈  ℝ20+1280 = ℝ1300

ESM-2 embeddings are loaded once from a shared HDF5 store keyed by normalized protein IDs; mutation positions are parsed from variant filenames during collation and mining.

Graph construction

SALAD-style hybrid topology

Each residue receives three classes of neighbors so the graph captures both local packing and long-range coupling without paying for all-pairs connectivity.

  • Index (16) — closest in residue sequence; primary-structure context.
  • Spatial (16) — k-nearest by Cα Euclidean distance.
  • Random long-range (16) — sampled with P(j) ∝ 1/d3 via a Gumbel top-k trick; preserves rare allosteric edges.
Edge feature vector — 20-D

Δseq

Why hybrid? DMS variants perturb both local packing and non-local couplings. Pure local edges miss compensatory long-range patterns; dense graphs blow up memory. The SALAD mix gives a sparse but information-rich compromise. A Graphein k-NN mode is retained as ablation (edge_attr_dim = 1).

Encoder

Residual GATv2 with edge conditioning

Backbone

2× GATv2 + residuals

Each layer: edge-MLP → GATv2 attention → LayerNorm → residual → ELU. Heads = 4, hidden = 128, projection = 256, ℓ₂-normalized.

Heads

Global · Local · GO

Global: mean-pool → MLP → 256-D. Local: node features at the mutated residue → MLP → 256-D. Phase 0: separate MF / BP / CC heads on top of the global embedding.

Phase 0 — GO pretraining

Function-aware initialization

Three ontology-specific heads share the encoder backbone and are trained on semantic-similarity triplets:

GO triplet loss · ontology o ∈ {MF, BP, CC} ℒGO,o = max( 0,  d(zao, zpo) − d(zao, zno) + mgo )

Active ontology losses are averaged per step. A weighted ratio sampler defaults to MF : BP : CC = 0.5 : 0.3 : 0.2 so no ontology dominates the gradient. The best phase-0 checkpoint warm-starts phase 1/2.

Phase 1 / 2 — DMS specialization

Joint global + local contrastive loss

Streaming semi-hard mining (global). For every anchor we evaluate all candidate negatives in fixed-size GPU chunks, classify them with the live margin, then keep up to 10 hard / semi-hard ones per family — bounded memory, high signal.

Global triplet loss with mined n* ℒg = max( 0,  d(zag, zpg) − d(zag, zn*g) + m )

Local contrastive (mutation site). Pairs the anchor's residue embedding at the mutated position with both its benign and its hardest pathogenic counterpart:

Local site contrastive loss · B = batch size ℒl = (1 / 2B) · Σi [ yi · di2 + (1 − yi) · max(0, m − di)2 ]
Combined objective ℒtotal = ½ · ( ℒg + ℒl )

Mining bookkeeping. Per-epoch hard / semi-hard / easy counts and the qualifying ratio are logged for both the global mined negatives and the local mutation-site distances, making loss-landscape diagnosis explicit.

Decoder · variant scoring

Multi-label GO classifier & Δscore readout

The encoder output is fed into a supervised feed-forward decoder that produces, for every relevant GO term, a sigmoid probability σ(·) ∈ [0, 1]. A dedicated "zero-function" output unit captures variants with complete loss of function.

Training

Native proteins only

Three independent decoders for MF / BP / CC, each trained on Swiss-Prot manually-curated GO annotations. Variant proteins are never seen during training — kept exclusively for inference.

Splits & loss

UniRef50 & BCE

Train / val / test splits cluster proteins at UniRef50 identity to prevent leakage with the encoder's DMS split. Loss: per-term binary cross-entropy.

Variant effect via decoder Δ-score · per GO term g Δg(WT, var) = σg( D(zWT) ) − σg( D(zvar) )

Sweeping a per-term decision threshold gives an AUPRC-optimal operating point. Reporting metrics: Acc · P · R · F1 · MCC · AUROC · AUPRC. The full pipeline ships as a CLI tool and a Hugging Face Space with two paths: (i) structure → predicted GO functions, (ii) WT vs. variant comparison with ClinVar lookup.

Take-away. By coupling a function-aware contrastive encoder (GO + DMS) with a supervised multi-label GO decoder, ContVAR moves variant interpretation from a binary "damaging / neutral" verdict to per-GO-term gain / loss percentages — the level of granularity needed for clinical and protein-engineering use.

Planned ablations (encoder & decoder, supported by config flags):

  • SALAD vs. Graphein k-NN edges; Phase 0 on / off; global-only vs. local-only vs. joint losses; streaming vs. random mining.
  • Decoder baseline: raw ESM-2 embeddings vs. ContVAR embeddings — fairness check that ESM has converged before comparison.
  • UniRef50 test split intersection with the DMS family split monitored to keep variant-side and native-side leakage-free.

Deliverables. Open-source repository, pretrained encoder + per-ontology decoder weights, large-scale function loss / gain tables for ClinVar & UniProt Humsavar, a CLI tool, and a Hugging Face Space GUI (Streamlit / Gradio).
